# SmartHire Project Interview Questions & Answers

---

## 1. Explain your SmartHire project.

SmartHire is a web-based recruitment platform tested as an end-to-end QA project. I performed manual testing for modules like Authentication, Candidate, Recruiter, Interview Scheduling, and Admin. I created test cases, bug reports, RTM, and also performed basic API testing using Postman.

---

## 2. What types of testing did you perform?

I performed:
- Functional Testing
- UI Testing
- Regression Testing
- Smoke Testing
- Basic API Testing

---

## 3. Which tools did you use in the project?

I used:
- Excel
- Postman
- GitHub

---

## 4. How did you report bugs?

I documented bugs in an Excel bug report sheet including:
- Bug ID
- Steps to reproduce
- Expected result
- Actual result
- Severity
- Priority
- Status

---

## 5. What APIs did you test?

I tested APIs related to:
- Login
- Registration
- Job application
- Job listing
- Interview scheduling

using Postman.

---

## 6. Difference between severity and priority?

Severity defines how serious the defect is technically.

Priority defines how urgently the defect should be fixed.

---

## 7. What is RTM?

RTM stands for Requirement Traceability Matrix. It is used to map requirements with test cases to ensure all requirements are covered during testing.

---

## 8. What challenges did you face during the project?

One challenge was designing proper negative test cases and validating different workflows like duplicate job applications and access restrictions.

---

## 9. Why is API testing important?

API testing helps validate backend functionality, response codes, request validation, and data flow between systems.

---

## 10. What did you learn from this project?

I learned:
- Test case design
- Defect reporting
- Requirement mapping
- Basic API testing
- End-to-end QA workflow
